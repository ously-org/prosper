# Product Guidelines: Prosper

## 1. Visual Identity
Prosper's visual identity is defined by a modern, high-contrast, and data-rich aesthetic that prioritizes clarity and a professional feel.

- **Theme:** Dark Mode First. A sleek, high-contrast dark interface tailored for developers and power users.
- **Color Palette:** A "Financial" palette utilizing deep greens, subtle blues, and a range of neutral grays to convey stability and precision.
- **Layout Philosophy:** Linear Simplicity. Drawing inspiration from the simplicity of Quicken, the UI should feel uncluttered and straightforward, despite the underlying complexity.
- **Components:** Leveraging `shadcn/ui` and `Radix UI` for a modern, consistent, and accessible component library.

## 2. User Experience (UX) Principles
- **Simplicity Above All:** Every feature should be easy to discover and use, even for complex financial modeling.
- **High Interactivity:** Prioritize responsiveness and smooth transitions. Every action (e.g., adding a node, forking a branch) should feel instantaneous and fluid.
- **Visual Hierarchy:** Use clear visual cues to guide users through the node-based branching tree, ensuring the "main" path and alternative scenarios are easily distinguishable.
- **Efficient Task Flow:** Design workflows that minimize friction, allowing users to quickly iterate on their financial plans and see the results immediately.

## 3. Interaction Design
- **Standardized Components:** Use consistent UI patterns across the entire application for a predictable user experience.
- **Keyboard-Driven Workflows:** Provide extensive keyboard shortcut support for power users who want to navigate and modify their plans without leaving the keyboard.
- **Instant Feedback:** Provide immediate visual and textual summaries as users modify their financial models. The outcome table should update in real-time.

## 4. Prose Style
- **Modern Professional:** Use language that is professional, precise, and approachable. Avoid excessive jargon while maintaining a level of technical accuracy suitable for the target audience.
- **Conciseness:** Keep UI labels and documentation brief and to the point.
